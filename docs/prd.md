## 1. 产品概述与目标

### 1.1 产品背景

在教育信息化快速发展的背景下，传统的手工记录和管理学生成绩方式已无法满足现代教学管理的需求。教师和教务人员需要一个高效、准确的成绩管理系统来提升工作效率，减少人为错误。

### 1.2 产品定义

**学生成绩管理系统**是一款面向学校教务管理人员和教师的Web应用系统，提供命令行工具和Web前端界面，用于实现学生信息的集中管理和成绩的自动化处理与统计分析。

### 1.3 目标用户

| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| **教务管理员** | 负责学生学籍管理和成绩录入 | 学生信息维护、批量成绩录入 |
| **任课教师** | 负责特定科目的成绩管理 | 本科目成绩录入、统计分析 |
| **班主任** | 负责班级整体学业情况 | 班级成绩汇总、排名统计 |
| **学生** | 查看个人成绩和排名 | 成绩查询、个人统计 |
| **家长** | 查看学生成绩情况 | 成绩查询、学习报告 |

### 1.4 产品目标

| 目标类型 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| **效率目标** | 减少成绩处理时间 | 相比手工方式提升 80% 以上 |
| **准确目标** | 降低数据录入错误率 | 错误率 < 0.1% |
| **易用目标** | 降低学习成本 | 新用户 30 分钟内上手 |

### 1.5 核心价值

1. **数据集中化：** 统一管理学生信息和成绩数据，避免数据分散
2. **操作自动化：** 自动计算统计指标，减少人工计算
3. **查询便捷化：** 多维度快速查询，秒级响应
4. **报表智能化：** 自动生成统计报表，辅助教学决策

## 2. 功能需求列表

### 2.1 功能优先级说明

| 优先级 | 标记 | 说明 |
|-------|------|------|
| P0 | 🔴 | 核心功能，必须实现 |
| P1 | 🟡 | 重要功能，应该实现 |
| P2 | 🟢 | 增强功能，可以实现 |

### 2.2 功能需求详细列表

#### 模块一：学生信息管理（F-STD）

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|
| F-STD-001 | 添加学生 | 🔴 P0 | 录入新学生基本信息 | 学号、姓名、性别、班级、入学年份 | 学生记录 |
| F-STD-002 | 删除学生 | 🔴 P0 | 删除已有学生记录 | 学号 | 操作结果 |
| F-STD-003 | 修改学生 | 🔴 P0 | 更新学生基本信息 | 学号、待修改字段 | 更新后记录 |
| F-STD-004 | 查询学生 | 🔴 P0 | 按条件查询学生信息 | 学号/姓名/班级 | 学生列表 |
| F-STD-005 | 学生列表 | 🟡 P1 | 查看所有学生信息 | 分页参数 | 分页学生列表 |

**业务规则：**
- 学号为唯一标识，不可重复
- 学号格式：4位年份 + 4位序号（如：20260001）
- 姓名长度限制：2-20个字符
- 班级命名格式：年级+班级（如：三年一班）

#### 模块二：成绩录入（F-GRD）

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|
| F-GRD-001 | 单条成绩录入 | 🔴 P0 | 录入单个学生的单科成绩 | 学号、科目、分数、考试类型 | 成绩记录 |
| F-GRD-002 | 批量成绩录入 | 🔴 P0 | 批量录入某科目的成绩 | 科目、考试类型、成绩列表 | 录入结果统计 |
| F-GRD-003 | 成绩修改 | 🟡 P1 | 修改已录入的成绩 | 成绩ID、新分数 | 更新后记录 |
| F-GRD-004 | 成绩删除 | 🟡 P1 | 删除错误的成绩记录 | 成绩ID | 操作结果 |

**业务规则：**
- 分数范围：0-100分（支持小数点后1位）
- 考试类型：期中、期末、月考、单元测试
- 科目列表：语文、数学、英语、物理、化学、生物、历史、地理、政治
- 同一学生同一科目同一考试类型只能有一条记录

#### 模块三：成绩查询（F-QRY）

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|
| F-QRY-001 | 按学生查询 | 🔴 P0 | 查询指定学生的所有成绩 | 学号/姓名 | 该学生全部成绩 |
| F-QRY-002 | 按班级查询 | 🔴 P0 | 查询指定班级的成绩 | 班级名称 | 班级成绩列表 |
| F-QRY-003 | 按科目查询 | 🔴 P0 | 查询指定科目的成绩 | 科目名称 | 科目成绩列表 |
| F-QRY-004 | 组合条件查询 | 🟡 P1 | 多条件组合查询 | 班级+科目+考试类型 | 筛选后成绩列表 |

#### 模块四：成绩统计（F-STA）

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|
| F-STA-001 | 平均分统计 | 🔴 P0 | 计算指定范围的平均分 | 班级/科目/考试类型 | 平均分 |
| F-STA-002 | 最高分统计 | 🔴 P0 | 查询指定范围的最高分 | 班级/科目/考试类型 | 最高分及学生 |
| F-STA-003 | 最低分统计 | 🔴 P0 | 查询指定范围的最低分 | 班级/科目/考试类型 | 最低分及学生 |
| F-STA-004 | 及格率统计 | 🔴 P0 | 计算及格率（≥60分） | 班级/科目/考试类型 | 及格率百分比 |
| F-STA-005 | 综合统计报告 | 🟡 P1 | 生成综合统计报表 | 班级/科目 | 完整统计报告 |

**统计指标定义：**

```
平均分 = 所有成绩总和 / 成绩数量
及格率 = (及格人数 / 总人数) × 100%
优秀率 = (优秀人数(≥90分) / 总人数) × 100%
```

#### 模块五：成绩排序（F-SRT）

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|
| F-SRT-001 | 单科成绩排序 | 🔴 P0 | 按单科成绩排序 | 科目、排序方式 | 排序后列表 |
| F-SRT-002 | 总分排序 | 🟡 P1 | 按总分排序 | 考试类型 | 排序后列表 |
| F-SRT-003 | 班级排名 | 🟡 P1 | 班级内成绩排名 | 班级、科目 | 排名列表 |
| F-SRT-004 | 年级排名 | 🟢 P2 | 年级内成绩排名 | 科目、考试类型 | 排名列表 |

**排序规则：**
- 默认降序排列（分数从高到低）
- 支持升序/降序切换
- 分数相同时按学号升序排列

#### 模块六：前端页面（F-FE）

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|
| F-FE-001 | 学生信息管理页面 | 🔴 P0 | 学生信息的完整CRUD界面 | 用户操作 | 数据列表、表单、操作结果 |
| F-FE-002 | 成绩录入页面 | 🔴 P0 | 支持单条和批量成绩录入 | 成绩数据 | 录入结果、进度反馈 |
| F-FE-003 | 成绩查询页面 | 🔴 P0 | 多维度成绩查询界面 | 查询条件 | 成绩列表、筛选结果 |
| F-FE-004 | 统计分析页面 | 🟡 P1 | 图表展示与排名统计 | 统计维度 | 可视化图表、排名表格 |
| F-FE-005 | 响应式布局 | 🟡 P1 | 适配移动端和桌面端 | 屏幕尺寸 | 自适应界面布局 |

**前端页面详细需求：**

**F-FE-001 学生信息管理页面：**
- 学生列表展示（分页、排序、搜索）
- 添加学生表单（表单验证、学号格式校验）
- 编辑学生信息（模态框或独立页面）
- 删除确认对话框
- 批量操作支持（批量删除、导出）

**F-FE-002 成绩录入页面：**
- 单条成绩录入表单（学生选择、科目选择、分数输入）
- 批量导入功能（CSV文件上传、模板下载）
- 录入进度显示（批量录入时）
- 成绩验证反馈（实时校验分数范围）
- 重复录入警告提示

**F-FE-003 成绩查询页面：**
- 按学生查询（学号/姓名搜索）
- 按班级查询（班级下拉选择）
- 按科目查询（科目多选）
- 组合条件筛选（班级+科目+考试类型）
- 查询结果导出（CSV/Excel）

**F-FE-004 统计分析页面：**
- 图表类型：柱状图、折线图、饼图、雷达图
- 统计维度：班级、科目、考试类型
- 排名展示：单科排名、总分排名、班级排名、年级排名
- 数据对比：不同考试类型成绩对比
- 统计报告导出（PDF格式）

**F-FE-005 响应式设计：**
- 移动端适配（手机、平板）
- 触摸友好的交互设计
- 自适应表格布局
- 移动端导航菜单
- 图表自适应缩放

**技术实现要求：**
- 前端框架：Vue 3 / React 18（二选一）
- UI组件库：Element Plus / Ant Design
- 图表库：ECharts / Chart.js
- 状态管理：Pinia / Redux
- HTTP客户端：Axios
- 构建工具：Vite

## 4. 验收标准

### 4.1 功能验收标准

#### AC-001: 学生信息管理验收

```
场景: 添加学生信息
  假设 系统已启动
  当 用户输入有效的学号、姓名、班级信息
  则 系统成功创建学生记录
  并且 返回创建成功的提示

场景: 学号重复校验
  假设 学号"20260001"已存在
  当 用户尝试添加相同学号的学生
  则 系统拒绝创建
  并且 提示"学号已存在"
```

#### AC-002: 成绩录入验收

```
场景: 单条成绩录入
  假设 学号"20260001"存在
  当 用户录入该学生语文成绩85分
  则 系统成功保存成绩记录
  并且 显示保存成功

场景: 分数范围校验
  当 用户输入分数150（超出范围）
  则 系统拒绝保存
  并且 提示"分数必须在0-100之间"
```

#### AC-003: 成绩查询验收

```
场景: 按学生查询
  假设 学号"20260001"有多科成绩
  当 用户查询该学生成绩
  则 系统返回该学生所有科目的成绩列表
  并且 显示科目名称、分数、考试类型
```

#### AC-004: 成绩统计验收

```
场景: 平均分统计
  假设 班级有30名学生数学成绩
  当 用户查询班级数学平均分
  则 系统返回正确的平均分（精确到小数点后2位）

场景: 及格率统计
  假设 30人中25人及格
  当 用户查询及格率
  则 系统返回83.33%
```

#### AC-005: 成绩排序验收

```
场景: 单科成绩排序

  当 用户按数学成绩降序排序

  并且 相同分数按学号升序排列
```

## 附录

### A. 数据模型定义

```python
# 学生信息模型
class Student:
    student_id: str      # 学号（主键）
    name: str            # 姓名
    gender: str          # 性别
    class_name: str      # 班级
    enrollment_year: int # 入学年份

# 成绩信息模型
class Score:
    score_id: int        # 成绩ID（主键）
    student_id: str      # 学号（外键）
    subject: str         # 科目
    score: float         # 分数
    exam_type: str       # 考试类型
    exam_date: date      # 考试日期
```

### B. 术语表

| 术语 | 定义 |
|-----|------|
| 学号 | 学生唯一标识符 |
| 考试类型 | 期中/期末/月考/单元测试 |
| 及格 | 分数 ≥ 60 |
| 优秀 | 分数 ≥ 90 |
| 响应式设计 | 界面自适应不同屏幕尺寸 |
| 组件化 | 将UI拆分为可复用的独立组件 |
